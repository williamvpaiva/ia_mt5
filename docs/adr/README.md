Architecture Decision Records (ADRs)
-----------------------------------

- Este repositório define decisões arquiteturais relevantes para o projeto, acompanhando o histórico de escolhas e suas consequências.
- Cada ADR tem um número único (ADR-001, ADR-002, etc.), um título, status (Proposto, Aceito, etc.), contexto, a decisão tomada, consequências e alternativas consideradas.
- Estruturado para facilitar revisões rápidas e rastreabilidade entre frontend e backend.

Guia rápido para escrever ADRs:
- Título claro e conciso.
- Contexto: qual problema/necessidade levou a esta decisão.
- Decisão: o que foi decidido e por quê.
- Consequências: impactos, migrações, custos, riscos, métricas.
- Alternatives considered: opções avaliadas (inclua por que foram rejeitadas).
- Status: Proposto | Aceito | Implementado | Rejeitado.

Formato sugerido para ADRs (exemplo):
- Arquivo ADR-001-frontend-architecture.md com seções: Context, Decision, Consequences, Status, Alternatives.
